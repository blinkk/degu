Optional noInterpolation

noInterpolation: undefined | false | true

Inherited from interpolateSettings.noInterpolation

  • Defined in interpolate/multi-interpolate.ts:46

Optional - noInterpolation property. This defaults to false and is used for very specific cases in which you want to specify pass through string values. The main uses case now is to pass through values like 'hidden' or 'none' to that cssVarInterpolate can add these properties to specific from/to ranges. See below for more.

Optional visibilityId

visibilityId: undefined | string

Inherited from interpolateSettings.visibilityId

  • Defined in interpolate/multi-interpolate.ts:62

An optional visibility id to pass to associate opacity and visibility. See https://github.com/blinkkcode/degu/issues/65.

This option is mainly used for css-var-interpolate to create a visibility css variable that sets to hidden when the related css variable (opacity) is set to 0.

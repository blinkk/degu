Class ScrollRenderFix

What this class does is, it eats the window.wheel event and eats the scroll. It allows the rendering to catch up and then once it is done, it reapplies the scroll to the document.

While this sounds counter-intuitive, it allows rendering to catchup and can smooth animations.

In short, if you have scroll tied animations or intense animations, this can help fix the issue.

Thanks to Angus and Eric for this tip.

Since this issue is chrome specific, you might want scope it to only chrome. Usage:

if (is.chrome()) {
  new ScrollRenderFix();
}

To take full advantage, use toolbox mutate or degu.read / writes.


const raf = new Raf();

raf.read(()=> {
  // do some reading
})

raf.writing(()=> {
  // do some writing
})
